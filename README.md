# kind-Gatekeeper
Этот проект представляет собой автоматизированную инфраструктуру для развёртывания локального Kubernetes-кластера с помощью Terraform и kind, а также последующей установки OPA Gatekeeper для обеспечения политики безопасности и управления конфигурациями.

Gatekeeper — это механизм контроля политик в Kubernetes.
Он проверяет все создаваемые ресурсы и отклоняет те, которые нарушают заданные правила. Используется для обеспечения безопасности, стандартизации и единообразных требований в кластере.

ConstraintTemplate
ConstraintTemplate описывает логику проверки в виде Rego-правил.Он определяет новый тип политики (CRD) и задаёт, что именно должно проверяться:
например, запрет привилегированных контейнеров или запрет тега latest.

Constraint
Constraint включает политику в работу.Определяет к каким ресурсам применять проверку, например:
только к Pod,
только в определённом namespace,
ко всему кластеру.

Итоговая схема
ConstraintTemplate — что проверять
Constraint — где применять
Gatekeeper — выполняет проверку при создании ресурсов
